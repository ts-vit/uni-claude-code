**Status:** ✅ SHIPPED 2026-05-18

## Overview

UX-полировка существующих экранов чата. Закрыла два пользовательских pain-point: переписка Claude терялась при переключении view или активного проекта (фактически unmount + создание новой `ChatPanel`), и нигде не было видно постоянно — с какой моделью идёт диалог, какой `session_id` соответствует логам в `~/.claude/projects/`, и сколько токенов уже потрачено в текущей сессии. Также добавлена UI-кнопка Clear (эквивалент текстовой команды `/clear`). Все изменения — frontend-only (~10 файлов компонентов чата); backend, `claude-code-core` и вендорированные пакеты не трогались.

## Phase Details

### Phase 4: Chat Persistence

**Goal:** Переписка во всех вкладках чата сохраняется при любых навигационных переключениях (view ↔ view, project ↔ project в пределах `maxOpenProjects`) — состояние не теряется ни визуально, ни в DOM, ни в памяти React.
**Depends on:** Nothing (первая фаза v1.1, опирается на shipped v1.0)
**Requirements:** PERSIST-01, PERSIST-02

**Success Criteria:**
1. Пользователь начинает чат с Claude в проекте A, переключается из view `main` в `settings` / `files` / `diff` / `history` / `claude-md` / `pipeline` и обратно — все сообщения, состояние стриминга и `session_id` видны без потерь.
2. Пользователь работает с двумя-тремя одновременно открытыми проектами (`ui.maxOpenProjects = 3`), переключает активный проект через сайдбар — переписка в каждом проекте сохраняется при возврате к нему.
3. Активная Claude-сессия (`isRunning === true`) продолжает принимать `claude-event` и накапливать сообщения в фоне, даже когда соответствующий проект/view не активен.
4. Идентификаторы вкладок (`panelId`) и привязанные runner'ы на бэкенде остаются неизменными при навигации — никаких «Session already running»-ошибок.

### Phase 5: Chat Visibility & Controls

**Goal:** В каждом моменте чата видно, с какой моделью идёт диалог, какой `session_id` корреспондирует с логами Claude и сколько токенов потрачено; базовая операция `/clear` доступна через явную кнопку в UI, а не только через текстовую команду.
**Depends on:** Phase 4 (StatusBar и ChatPanel должны переживать навигацию, иначе показывать постоянные метаданные бессмысленно)
**Requirements:** VIS-01, VIS-02, VIS-03, UI-01

**Success Criteria:**
1. Пользователь видит имя активной модели Claude (например `claude-sonnet-4-6`) постоянно в StatusBar для каждой запущенной вкладки чата.
2. Пользователь видит `session_id` текущей Claude-сессии в StatusBar (8-символьный prefix + Mantine `CopyButton`), может скопировать полный UUID.
3. Пользователь видит накопленное token usage текущей сессии в StatusBar — отдельно input / output / cache (cache_creation + cache_read) с Tooltip-breakdown.
4. Пользователь нажимает явную кнопку в шапке `ChatPanel` и полностью очищает переписку (эквивалент текстовой `/clear`).
5. Метаданные StatusBar корректно обновляются при пересоздании сессии — нет «прилипших» значений.

## Milestone Summary

**Key Decisions:**

- Keep-mounted рендер вместо conditional unmount — `display: none/flex` на sibling-блоках — единственный способ физически сохранить React tree при навигации; DB-persistence отложена.
- До 3 `DualPanelLayout` одновременно в DOM — `ui.maxOpenProjects = 3` достаточна; больше — риск перфоманса при многих xterm-инстансах.
- Token usage как абсолютное число (не % от лимита) — расчёт «% от контекста» требует справочника per-model лимитов; отложено в `VIS-CTX-01`.
- Ref-based comparison для reset accumulator (CR-01 fix) — impure `setState` в `case "system"` нарушал чистоту event-handler'а; ref-сравнение даёт ту же семантику без side-effects в render-path.

**Issues Resolved:**

- Переписка терялась при `view ↔ view` и `project ↔ project` переключениях (PERSIST-01, PERSIST-02).
- Имя модели было видно только в одноразовом system-сообщении (VIS-01).
- `session_id` нигде не отображался для корреляции с логами Claude (VIS-02).
- Не было постоянной видимости накопленных токенов сессии (VIS-03).
- `/clear` был доступен только через текстовую команду (UI-01).

**Issues Deferred (technical debt):**

- **WR-02: terminal refit на view-switch** — после keep-mounted рендера `triggerTerminalRefit()` не вызывается при `display:none → display:flex`; xterm/FitAddon может показывать обрезанные строки после resize окна при невидимой панели. Решение пользователя: «старый баг пока править не будем». Трекается в `.planning/todos/pending/wr-02-terminal-refit-view-switch.md`.

**Out-of-Scope (deferred for future milestones):**

- `PERSIST-DB-01` — DB-персистентность переписки между запусками приложения (SQLite + миграция схемы + sync с панелями).
- `PERSIST-CONTINUE-01` — интеграция с `--continue` Claude CLI для подгрузки прошлых сессий из CLI-кэша.
- `VIS-CTX-01` — context-window % с per-model лимитами Claude (200k / 1M).
- `UI-TAB-01` — осмысленные лейблы табов (модель / первый промпт) вместо «Session 1/2/3».
- `UI-CLOSE-01` — `ConfirmModal` при попытке закрыть таб с `status === "running"`.
- `UI-MODE-01` — кликабельный mode-badge в шапке `ChatPanel` для переключения architect/developer.
