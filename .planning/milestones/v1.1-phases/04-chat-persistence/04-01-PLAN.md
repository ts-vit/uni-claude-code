(3) Переписать тело `<AppShell.Main>` (текущие строки 245-302) так, чтобы:

- Корневой обёрточный `<div>` с `style={{ height: "calc(100vh - 50px)", overflow: "hidden" }}` сохраняется.
- Внутри этого `<div>` всегда (при `activeProject !== null`) рендерится **ОДИН** контейнер с рендером всех `openedProjects` через `.map` (БЕЗ фильтра `project.id === activeProject.id && view === "main"` — фильтр удалён, рендерим ВСЕ openedProjects одновременно). Видимость каждого проекта управляется CSS:
  - оборачивающий `<div key={project.id}>` получает `style={{ display: project.id === activeProject.id && view === "main" ? "flex" : "none", flex: 1, flexDirection: "column", height: "100%", overflow: "hidden" }}`.
  - Внутри этого div рендерится `<DualPanelLayout cwd={project.cwd} projectId={project.id} projectModel={project.model} projectPermissionMode={project.permissionMode} />` БЕЗ props `initialState` и `onStateChange`.
- Sibling-блок ПОВЕРХ (после в JSX, тот же родитель — sibling, не overlay через z-index согласно рекомендации D-04-03) рендерит компонент текущего view, КОГДА `view !== "main"`. Использовать прежнюю ladder-логику, но обёрнутую в один `<div>` с `style={{ display: view !== "main" ? "flex" : "none", flexDirection: "column", flex: 1, height: "100%", overflow: "hidden" }}`. Внутри ladder по `view`:
  - `view === "settings"` → `<SettingsPage />`
  - `view === "pipeline"` → `<PipelinePage initialProjectId={activeProject?.id} onProjectSelect={handleProjectSelect} />`
  - `view === "claude-md" && activeProject` → `<ClaudeMdEditor cwd={activeProject.cwd} />`
  - `view === "diff" && activeProject` → `<DiffViewer cwd={activeProject.cwd} initialFile={diffInitialFile} />`
  - `view === "files" && activeProject` → `<FileTreePanel cwd={activeProject.cwd} onFileSelect={(path) => { setDiffInitialFile(path); setView("diff"); }} />`
  - `view === "history" && activeProject` → `<HistoryPage projectId={activeProject.id} />`
  - Иначе (например `view === "settings"` или `view === "pipeline"` без `activeProject`) — внутри блока рендерится `null`. Для остальных view без `activeProject` ladder также падает в `null`.
- Когда `activeProject === null`, always-mounted блок с openedProjects-map не нужен (рендерим `null` или вовсе не рендерим этот блок — `openedProjects` в этом случае пустой). При `activeProject === null` И `view === "main"` показывается `<WelcomeScreen onCreateProject={() => setCreateModalOpened(true)} />`. Когда `activeProject === null` И `view === "settings"` или `view === "pipeline"` — view-блок видим через `display:flex`, WelcomeScreen скрыт.
- Структурно итог: корневой `<div style=height:calc(100vh - 50px)>` содержит ровно три sibling-блока:
  (a) main-блок: `{activeProject ? <div style={display:view==="main"?"flex":"none"}> openedProjects.map(<DualPanelLayout/>) </div> : null}` — хранит все ChatPanel живыми;
  (b) view-блок: `<div style={display:view!=="main"?"flex":"none"}> {view==="settings"?<SettingsPage/>:view==="pipeline"?<PipelinePage/>:...} </div>` — показывает компонент current non-main view;
  (c) welcome-блок: `{!activeProject && view === "main" ? <WelcomeScreen onCreateProject={() => setCreateModalOpened(true)}/> : null}` — рендерится только когда нет активного проекта и мы во view main.
